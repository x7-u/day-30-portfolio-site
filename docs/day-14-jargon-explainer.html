Jargon Explainer

Glossary for finance jargon in any document
STACKPython / Flask / DeepSeek FOCUSTraining and accessibility PORT:1014 TESTS41

About the project

Drop in a finance document; AI finds the jargon, explains it in plain English, builds a glossary.

Local-only Flask web app and CLI that takes any paragraph of dense financial text and produces a plain-English glossary, tuned to the document type. Bond prospectuses, fund factsheets, annual reports, earnings call transcripts, research notes -- paste anything and the matched jargon gets highlighted in marker-coloured strokes by category, with a click-through term card explaining each one.

At a glance

Glossary size
around 70 curated finance terms with aliases, across 13 categories
Inputs
Pasted financial text (free text or pick a bundled sample)
Outputs
Plain-English glossary + worked examples + sentence rewrites + new-jargon discovery
Model
DeepSeek V4 (deepseek-chat), one call per analysis (deterministic fallback if offline)
Cost
around $0.0005 per analysis

How it works

  • Glossary. glossary.py carries around 70 curated finance terms across 13 categories.
  • Jargon finder. jargon_finder.py builds one big OR-regex from all surface forms (canonical + aliases) with negative lookbehind / lookahead for non-letter / non-digit characters as word boundaries.
  • Doc-type classifier. doc_type.py is heuristic-only. Each document type has a list of (regex, weight) rules.
  • AI explainer. ai_explainer.py builds a prompt with the source text, predicted doc type + confidence, and the matched glossary entries (with their disambiguation hints), and asks DeepSeek V4 in...
  • Highlighter UI. Distinct from Day 7 (Bloomberg terminal), Day 8 (glass orbs), Day 9 (parchment atelier), Day 10 (magazine bold blocks), Day 11 (FT pink broadsheet), Day 12 (matte black drafting),...
  • Excel and CSV exports. openpyxl writes a 5-sheet workbook (Summary + Glossary + Hits + Rewrites + New Jargon).

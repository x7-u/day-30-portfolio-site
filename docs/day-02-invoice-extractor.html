Invoice Workbench

PDF invoice extraction to ledger CSV
STACKPython / Flask / pdf parsing / Anthropic FOCUSAccounts payable automation PORT:1002 TESTS33

About the project

Drag PDF or image invoices in; the AI extracts line items, totals and counterparty into a normalised ledger. Confidence scores and an audit trail per line.

Local-only Flask web app + CLI that uses Claude's multimodal vision API to extract structured data (vendor, dates, line items, totals) from invoice images and PDFs. Output is CSV (one row per line item) plus a multi-sheet Excel workbook. **Runs on your machine only - no third-party hosting.**

At a glance

Use case
Accounts-payable automation, invoice data capture from PDFs and images
Inputs
Invoice PDFs or images (jpg / png / webp); multi-page PDFs capped at 5 pages
Outputs
CSV (one row per line item), a multi-sheet Excel workbook, and a per-invoice confidence band and duplicate flag
Model
Claude Haiku 4.5 vision default, Sonnet 4.6 / Opus 4.7 selectable
Cost
around $0.003 to $0.005 per invoice on Haiku 4.5, $0 with --no-ai

How it works

  • Page rasteriser. pdf_loader.py turns each PDF or image page into image bytes with pypdfium2 and Pillow (EXIF-rotate, clamp to a 1568px edge).
  • Vision extraction. pipeline.py sends each page to Claude's vision API and gets strict JSON back: vendor, invoice number, dates, line items, subtotal, tax, currency and a confidence band.
  • Arithmetic validation. invoice_schema.py checks line items sum to subtotal and subtotal plus tax equals total; mismatches add warnings and downgrade confidence rather than silently correcting.
  • Duplicate ledger. ledger.py fingerprints every extraction (vendor, number, date, currency, total) and flags re-uploads as EXACT, SUSPICIOUS or FUZZY.
  • CSV + Excel output. one accountant-friendly row per line item, plus an Invoices / Line items / Summary workbook with an embedded chart and confidence pills.

AI integration

Claude extracts and normalises invoice fields

Limitations

  • AI proposes, you decide. Validation surfaces discrepancies but never silently corrects the model's numbers.
  • Multi-page PDFs are capped at 5 pages per file; over-cap files trigger a warning.
  • Generic CSV export. Direct Xero / QuickBooks / Sage import formats are post-30.
  • No on-device OCR fallback. Offline runs produce stub output; a Tesseract fallback is post-30.
  • Local-only. Loopback Flask dev server with no auth; production hosting is a later step.

Fun facts

  • Drag in a stack of PDF invoices and it pulls out the line items, totals and supplier into one tidy ledger, with a confidence score on every row.
  • Every extracted figure keeps an audit trail back to the invoice, so nothing is taken on trust.

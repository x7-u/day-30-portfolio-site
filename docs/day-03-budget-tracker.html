Budget vs Actual

Variance tracking with management commentary
STACKPython / Flask / openpyxl / Anthropic FOCUSBudget control and management reporting PORT:1003 TESTS15

About the project

Budget and actual side-by-side with per-row tolerance, direction-aware adverse vs favourable, and AI-drafted management commentary.

Local-only Flask web app and CLI for budget vs actual variance analysis with an AI-drafted management commentary in the voice of a financial controller. Drop in a workbook, walk away with the same pack a finance team would assemble manually for the month-end review. **Runs on your machine only, loopback bound, no third-party hosting.**

At a glance

Model
Claude Haiku 4.5 (default), Sonnet 4.6 / Opus 4.7 selectable
Cost
around $0.004 per analysis on Haiku 4.5

How it works

  • The Tracker. Drop zone front and centre, three bundled samples on the right.
  • AI Commentary. Pre-summarised pipe-delimited digest of headline numbers, top-5 adverse drivers, top-5 favourable drivers and the three rollups.
  • Excel Output. Six sheets single-period, seven multi-period.
  • Power BI Script. One-click Power Query M generator. Paste it into Power BI Desktop's blank query editor and you get the same long-format table the app uses, ready to model.
  • Comparison and History. Last 100 runs persisted to JSONL, FIFO eviction.
  • PowerPoint and JSON. Three-slide deck on demand: title, KPI tiles plus headline narrative, drivers and actions.

AI integration

Claude drafts the period commentary

Limitations

  • GBP only in the MVP. Non-GBP metadata is rejected with a clear remediation message.
  • Period labels are opaque strings. April-26, 2026-04 and Q1 26 all work; they sort by first-seen order.
  • AI commentary is a draft. The fallback when the API errors is a stub card with the error reason; the rest of the analysis still runs.
  • Power BI export is a Power Query script, not a .pbix file.
  • Run history caps at 100 with FIFO eviction. The JSONL store is fine for a single user; a finance team would want a shared store, which is post-30.
  • Single-flight on /api/analyse. A slow run blocks a second click.

Fun facts

  • It knows the difference between good and bad news: an overspend on costs is flagged red, but coming in under budget is celebrated, not punished.
  • Each line can carry its own tolerance, so a lumpy travel budget is not held to the same standard as fixed rent.

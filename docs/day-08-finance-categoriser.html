About the project

Drop in a bank statement; the engine categorises every transaction by merchant dictionary plus AI for edge cases.

Local-only Flask web app and CLI for taking a UK bank statement CSV and turning it into a categorised spend dashboard with a per-category pie chart, monthly trend, recurring subscription detector, and a savings rate.

At a glance

Model
DeepSeek V4 (deepseek-chat default), one batched call per statement
Cost
around $0.001 per analysis.

How it works

  • Multi-bank parser. statement_schema.py auto-detects the bank format from the CSV headers.
  • PDF parser. pdf_parser.py uses PyMuPDF's word-level extraction with Y-coordinate clustering, plus the page index in the cluster key so identical Y values on different pages do not collide.
  • Two-pass categoriser. categoriser.py runs the rule pass first against merchant_dict.py (200+ UK patterns).
  • Visual layer. All client-side. Inline-SVG donut chart with stroke-dasharray slice reveal.
  • Excel and exports. openpyxl writes a 5-sheet workbook (Summary + Categories + Transactions + Recurring + Monthly + Inputs) with the donut and monthly-trend PNGs embedded into Summary via BytesIO so...

AI integration

DeepSeek categorises ambiguous transactions

Fun facts

  • It sorts a bank statement into categories using a merchant dictionary first and only calls on AI for the genuinely odd ones, which keeps it fast and cheap.
  • Built for the messy reality of real statements, not a tidy textbook example.
